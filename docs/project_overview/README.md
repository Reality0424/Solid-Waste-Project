## ⚡ 5分钟快速入门

### 1️⃣ 理解项目 (2分钟)

**问题**: 芯片检测通过了,为什么还会坏?
```
传统方法:
  ✗ VDD=3.3V (OK) + IDD=120mA (OK) + VOH=2.95V (OK) + ... 
  → "全部OK,芯片合格"
  → 但客户用72小时后... 芯片坏了!

原因: 所有参数都临界合格,组合在一起就会雪崩
  - 噪声余裕压缩到危险值
  - 时序余量消耗殆尽
  - 热积累加速
```

**本项目的解决**: 
```
✓ 用GCN+GAT识别"多参数组合"导致的隐性故障
✓ 用Shapley值找出最关键的指标(从28项减到5-10项)
✓ 用PPO强化学习优化检测流程(检测时间减50%)
✓ 用CSS增量学习让系统自我完善
```

### 2️⃣ 了解数据 (2分钟)

我们准备了**1000个合成芯片样本**:

| 文件 | 行数 | 列数 | 内容 |
|-----|------|------|------|
| chip_baseline_data.csv | 1,000 | 28 | 每个芯片的初始参数 |
| chip_aging_curves.csv | 90,424 | 7 | 老化过程中的参数漂移 |
| chip_failure_labels.csv | 100 | 6 | 故障品的失效信息 |

**数据包含什么**:
- ✓ 物理检测 (6项: 平整度、翘曲度、划痕等)
- ✓ 电气参数 (15项: VDD、IDD、VOH、VOL、频率等)
- ✓ 功能测试 (3项: JTAG、边界扫描、功能测试)
- ✓ 时间序列 (老化过程中的参数变化)
- ✓ 失效标注 (4种故障模式标记)

## ⚠️ 常见问题

**Q: 这些数据能直接用于生产吗?**  
A: 不能。这是概念验证(MVP)用的合成数据。生产前需要与企业真实数据混合并验证。

**Q: 如果企业永远无法提供数据?**  
A: 基于合成数据的模型也能部署,但准确率会有限制(~88-90%)。需要持续增量学习。

**Q: 1000个样本够吗?**  
A: 对于MVP验证框架足够。生产级通常需要10,000+。可通过修改脚本参数轻松扩展。

**Q: 怎么知道数据的逼真度?**  
A: 参数分布符合工程统计学,物理约束完全满足。但与真实数据的分布偏差需要后期验证。

**Q: 我怎么参与这个项目?**
