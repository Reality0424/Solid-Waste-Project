## 🔍 详细数据结构

### 1️⃣ chip_baseline_data_extended.csv (核心数据，1000个芯片)

**用途**: 支持 Phase 1-5 的所有分析  
**包含内容**: 39个字段，覆盖4个测试阶段

#### 字段分类

**A. 芯片信息**
```
chip_id                      # 芯片唯一标识 (CHIP_00000 - CHIP_00999)
production_date              # 生产日期 (2024-01-01 ~ 2024-12-31)
chip_model                   # 芯片型号 (MODEL_A, MODEL_B, MODEL_C)
```

**B. 外观测试参数** (Phase 1-2需要)
```
pin_flatness_deviation_um           # 引脚平坦度偏差 (μm)
solder_pad_oxidation_percent        # 焊盘氧化率 (%)
package_scratch_depth_um            # 封装划伤深度 (μm)
warpage_um                          # 翘曲量 (μm)
pin_coplanarity_percent             # 引脚共面性 (%)
package_size_deviation_percent      # 封装尺寸偏差 (%)
appearance_test_pass                # 外观测试是否通过 (0/1)
appearance_quality_score            # 外观质量评分 (0-100)
```

**C. DC电气测试参数** (Phase 2-3需要)
```
vdd_voltage                 # 电源电压 (V)
idd_static_ma              # 静态电流 (mA)
idd_dynamic_ma             # 动态电流 (mA)
voh_voltage                # 高电平输出电压 (V)
vol_voltage                # 低电平输出电压 (V)
vih_voltage                # 高电平输入电压阈值 (V)
vil_voltage                # 低电平输入电压阈值 (V)
iil_leakage_ua             # 输入漏电流 (μA)
iol_leakage_ua             # 输出漏电流 (μA)
dc_test_pass               # DC测试是否通过 (0/1)
dc_electrical_quality_score # DC质量评分 (0-100)
```

**D. AC电气测试参数** (Phase 2-3需要)
```
setup_time_ns              # 建立时间 (ns)
hold_time_ns               # 保持时间 (ns)
propagation_delay_ns       # 传播延迟 (ns)
rise_time_ns               # 上升时间 (ns)
fall_time_ns               # 下降时间 (ns)
fmax_mhz                   # 最大工作频率 (MHz)
ac_test_pass               # AC测试是否通过 (0/1)
ac_electrical_quality_score # AC质量评分 (0-100)
```

**E. 功能测试参数** (Phase 2-3需要)
```
jtag_scan_pass             # JTAG扫描是否通过 (0/1)
boundary_scan_pass         # 边界扫描是否通过 (0/1)
functional_test_pass       # 功能测试是否通过 (0/1)
functional_quality_score   # 功能质量评分 (0-100)
```

**F. 老化测试条件** (Phase 3-6需要)
```
aging_temperature_c        # 老化温度 (°C, 固定为85)
aging_voltage_v            # 老化电压 (V, ~3.3)
aging_frequency_mhz        # 老化频率 (MHz, ~250)
aging_duration_hours       # 老化时长 (小时, 24-168)
reliability_risk_level     # 可靠性风险等级 (low/medium/high)
```

#### 数据统计

```python
# Phase 1-2: 知识图谱 & 特征工程
- 500个实体示例: 芯片型号、参数名称、测试阶段等
- 关系类型: 外观→DC→AC→功能→老化 的因果关系
- 28个参数 + 11个衍生评分 = 39列

# Phase 3: 动态检测策略
- 测试流程: 外观(appearance) → DC(dc_electrical) 
  → AC(ac_electrical) → 功能(functional)
- 参数相关性: 外观质量影响DC，DC影响AC，AC影响功能

# Phase 5-6: 适配性预测
- 质量评分范围: 0-100
- 通过/失败标记: 0(失败)/1(通过)
```

### 2️⃣ chip_test_sequence.csv (测试序列，4000行)

**用途**: Phase 3 动态检测策略优化  
**结构**: 1000个芯片 × 4个测试阶段 = 4000行

**列信息**
```
chip_id                     # 芯片ID
test_stage                  # 测试阶段名称
                            #   - appearance (外观)
                            #   - dc_electrical (DC电气)
                            #   - ac_electrical (AC电气)
                            #   - functional (功能)
stage_order                 # 阶段顺序 (1-4)
test_cost_yuan              # 单阶段测试成本 (人民币)
test_time_minutes           # 单阶段测试时间 (分钟)
cumulative_cost_yuan        # 累计成本 (元)
cumulative_time_minutes     # 累计时间 (分钟)
```

#### 用途示例

```python
# Phase 3: 动态检测策略优化
# 问题: 是否可以改变检测顺序以降低成本/时间？

# 原始顺序成本统计:
固定顺序 (appearance → dc → ac → functional):
  总成本: 330-590元
  总时间: 145-270分钟

# PPO智能体的任务:
  状态: 当前检测进度、已识别的质量问题
  动作: 选择下一步检测 (优化顺序)
  奖励: 检测时间↓ 或 检测成本↓ (≥30%)
```

### 3️⃣ chip_failure_labels_extended.csv (故障标签，1000个故障样本)

**用途**: Phase 5-6 故障分析和适配性预测  
**规模**: 1000个故障样本（从1000个芯片中随机选择~10%失败）

**列信息**
```
chip_id                     # 故障芯片ID
failure_mode                # 故障模式 (4种)
                            #   - physical_damage (物理损伤)
                            #   - thermal_accumulation (热积累)
                            #   - noise_margin_insufficient (噪声裕度不足)
                            #   - multi_param_combined (多参数组合)
failure_time_hours          # 故障发生时间 (小时)
failure_phenomenon          # 故障表现 (故障症状描述)
root_cause                  # 根本原因 (参数降级分析)
predicted_mttf_hours        # 预测MTTF (平均故障间隔时间)
```

#### 故障模式分布

```
physical_damage               ~ 250个 (25%)
  症状: 物理连接断裂，器件断开
  修复: 无法修复，拒收

thermal_accumulation          ~ 250个 (25%)
  症状: 热失控，功耗激增，触发关闭保护
  修复: 热管理增强，降速运行

noise_margin_insufficient     ~ 250个 (25%)
  症状: 信号完整性失败，间歇性错误
  修复: 频率降速，电压调整

multi_param_combined          ~ 250个 (25%)
  症状: 时序违规，高频功能测试失败
  修复: 应用范围受限，频率上限
```

### 4️⃣ chip_compatibility_matrix.csv (适配性矩阵，4000行)

**用途**: Phase 5 适配性预测和应用场景匹配  
**结构**: 1000个芯片 × 4个应用场景 = 4000行

**列信息**
```
chip_id                     # 芯片ID
application_scenario        # 应用场景 (4种)
                            #   - high_frequency_computing (高频计算)
                            #   - power_efficient_device (低功耗设备)
                            #   - high_reliability_critical (高可靠性)
                            #   - consumer_electronics (消费电子)
compatibility_score         # 适配性评分 (0-100)
compatibility_level         # 适配性等级
                            #   - compatible (兼容, >70)
                            #   - marginal (边界, 50-70)
                            #   - incompatible (不兼容, <50)
```

#### 评分逻辑

```
高频计算 (high_frequency_computing):
  影响因素: fmax_mhz, propagation_delay_ns, ac_test_pass
  权重: 频率40% + 延迟30% + 功能30%

低功耗 (power_efficient_device):
  影响因素: idd_dynamic_ma, idd_static_ma, vdd_voltage
  权重: 动态电流35% + 静态电流35% + 电压稳定性30%

高可靠性 (high_reliability_critical):
  影响因素: reliability_risk_level, aging_duration_hours
  权重: 风险等级60% + 老化耐受力40%

消费电子 (consumer_electronics):
  影响因素: appearance_quality_score, functional_test_pass
  权重: 外观质量40% + 功能通过60%
```

### 5️⃣ chip_incremental_evolution.csv (增量学习数据，5000行)

**用途**: Phase 4 增量学习和模型自适应  
**结构**: 1000个芯片 × 5个更新步骤 = 5000行

**列信息**
```

update_step                 # 更新步骤 (1-5)
                            #   模拟模型随真实数据不断改进

idd_dynamic_ma              # 动态电流 (mA)
propagation_delay_ns        # 传播延迟 (ns)
fmax_mhz                    # 最大频率 (MHz)
model_accuracy              # 模型精度 (0.75 → 0.90)
model_version               # 模型版本 (v1.1 → v1.5)
```

#### 用途示例

```python
# Phase 4: 增量学习 (Continuous/Streaming Learning)
# 问题: 如何在不"灾难遗忘"的情况下，用真实数据改进模型?

初始模型 (v1.1): 精度 78%
  基于合成数据训练，存在域差异

新数据到达 (update_step 1-5):
  → 用CSS机制更新模型
  → v1.2: 精度 81%
  → v1.3: 精度 84%
  → v1.4: 精度 87%
  → v1.5: 精度 90%

约束:
  - 每次更新后，模型在旧数据上精度不下降
  - 自适应更新周期 ≤ 1周
```

# 1. 加载基线数据
import pandas as pd
df = pd.read_csv('synthetic_data/chip_baseline_data_extended.csv')

# 2. 提取外观→DC→AC→功能的参数

## 📌 注意事项

1. **数据是合成的**: 参数来自统计分布，不代表真实芯片数据
   - 用于算法开发和框架验证 ✅
   - 不能直接用于工业生产决策 ❌

2. **参数间相关性**: 已模拟上下游测试阶段的因果关系
   - 外观质量影响DC测试 → 影响AC测试 → 影响功能测试 ✅

3. **故障模型**: 基于通用的芯片老化理论
   - 物理损伤、热积累、噪声余度、参数组合 ✅

4. **升级数据的时机**:
   - 现在: 框架开发和算法验证 (Phase 1-3)
   - 9月: 补充真实企业数据 (Phase 4-6)
