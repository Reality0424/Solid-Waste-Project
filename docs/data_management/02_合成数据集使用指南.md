## 一、数据集概览

### 1.1 三个核心文件

#### **文件1: chip_baseline_data.csv** (基础参数数据)
- **行数**: 1,000行（每行代表1个芯片）
- **列数**: 28列
- **用途**: 芯片初始状态的完整电气和物理参数
- **文件大小**: ~3-5 MB

**关键字段**:
```
chip_id               : 芯片唯一标识符 (CHIP_00000 ~ CHIP_00999)
production_date       : 生产日期 (2024-01-01 ~ 2024-12-31)
chip_model            : 芯片型号 (MODEL_A / MODEL_B / MODEL_C)

【物理参数】6项
pin_flatness_deviation_um          : 引脚平整度偏差 (0-50μm)
solder_pad_oxidation_percent       : 焊盘氧化程度 (0-100%)
package_scratch_depth_um           : 封装划痕深度 (0-100μm)
warpage_um                         : 翘曲度 (0-200μm)
pin_coplanarity_percent            : 引脚共面度 (80-100%)
package_size_deviation_percent     : 尺寸偏差 (±5%)

【直流电气参数】9项
vdd_voltage                        : 供电电压 (V)
idd_static_ma                      : 静态功耗 (mA)
idd_dynamic_ma                     : 动态功耗 (mA)
voh_voltage                        : 输出电压高 (V)
vol_voltage                        : 输出电压低 (V)
vih_voltage                        : 输入阈值高 (V)
vil_voltage                        : 输入阈值低 (V)
iil_leakage_ua                     : 输入漏电流 (μA)
iol_leakage_ua                     : 输出漏电流 (μA)

【交流参数】6项
setup_time_ns                      : 建立时间 (ns)
hold_time_ns                       : 保持时间 (ns)
propagation_delay_ns               : 传输延迟 (ns)
rise_time_ns                       : 上升时间 (ns)
fall_time_ns                       : 下降时间 (ns)
fmax_mhz                           : 最高工作频率 (MHz)

【功能测试】3项
jtag_scan_pass                     : JTAG扫描 (1=通过, 0=失败)
boundary_scan_pass                 : 边界扫描 (1=通过, 0=失败)
functional_test_pass               : 功能测试 (1=通过, 0=失败)

【老化条件】4项
aging_temperature_c                : 老化温度 (°C)
aging_voltage_v                    : 老化电压 (V)
aging_frequency_mhz                : 老化频率 (MHz)
aging_duration_hours               : 老化时长 (小时)

【标签】1项
failure_status                     : 故障标签 (1=正常品, 0=故障品)
failure_mode                       : 故障模式 (normal/physical_damage/noise_margin_insufficient/thermal_accumulation/multi_param_combined)
```

#### **文件2: chip_aging_curves.csv** (老化过程时间序列数据)
- **行数**: 90,424行（每个芯片24-168小时，分钟级采样）
- **列数**: 7列
- **用途**: 芯片在老化过程中的参数漂移轨迹
- **文件大小**: ~2-3 MB

**关键字段**:
```
chip_id                    : 芯片标识 (对应baseline数据)
aging_hour                 : 老化时间点 (0-168小时)
vdd_voltage                : 供电电压随时间的变化
idd_dynamic_ma             : 动态功耗随时间的变化
voh_voltage                : 输出高电压随时间的变化
vol_voltage                : 输出低电压随时间的变化
propagation_delay_ns       : 传输延迟随时间的变化

应用：
✓ 识别参数漂移的非线性特征
✓ 检测隐性故障的加速劣化现象
✓ 训练时间序列预测模型 (LSTM/GRU)
```

#### **文件3: chip_failure_labels.csv** (失效标注数据)
- **行数**: 100行（对应100个故障品）
- **列数**: 6列
- **用途**: 失效品的失效现象、根本原因、预测寿命
- **文件大小**: ~50-100 KB

**关键字段**:
```
chip_id                  : 故障芯片标识
failure_mode             : 失效模式 (4种类型)
failure_time_hours       : 失效时刻 (老化过程中的哪一小时)
failure_phenomenon       : 失效现象描述 (英文描述)
root_cause               : 根本原因分析
predicted_mttf_hours     : 预测平均失效时间
```

## 二、数据质量特性

### 2.1 符合工程实际的特性

| 特性 | 实现 | 验证 |
|-----|------|------|
| **参数分布** | 正态分布+指数分布，符合工程统计学 | ✓ |
| **物理约束** | VOH>VOL, VIH>VIL, VDD>VOH, 0>VOL等 | ✓ |
| **多型号覆盖** | 3种芯片型号，参数特性不同 | ✓ |
| **时间序列连续性** | 老化曲线光滑，参数漂移符合物理规律 | ✓ |
| **失效模式多样** | 4种典型失效模式：物理、噪声、热、多参数组合 | ✓ |

### 2.2 数据集的关键创新

**创新点1: 隐性故障的显式表现**
```
传统数据：CHIP_A 全部参数 PASS → 标记为"正常"

本数据集：
- CHIP_A 虽然所有单指标都在范围内
- 但 VOH=2.85V(临界) + VOL=0.35V(临界) + VIH=2.32V(临界) + VIL=0.88V(临界)
- 组合导致噪声余裕=0.13V，远低于安全值0.5V
- 标记为"noise_margin_insufficient"故障品
```

**创新点2: 客观失效时间点**
```
实际客户反馈是"用了N小时后坏了"
本数据集提供：
- failure_time_hours: 71 (告诉模型这个芯片在老化71小时时会失效)
- 对应在 chip_aging_curves.csv 中的具体参数漂移轨迹
- 模型可以学习"什么样的参数漂移轨迹通常导致失效"
```

**创新点3: 时间序列驱动的模式识别**
```
正常品的老化曲线：
- VOH: 2.966 → 2.965 (漂移-0.03%)
- VOL: 0.077 → 0.077 (基本不变)
- 延迟: 4.669 → 4.670 (缓慢上升)

故障品的老化曲线 (热积累型):
- VOH: 2.966 → 2.940 (漂移-0.88%, 快速下降)
- VOL: 0.077 → 0.130 (快速上升)
- 延迟: 4.669 → 5.200 (快速增加)

模型学习这些曲线模式，可预测新芯片会不会失效
```

#### **子任务1: 多维关系建模与基础语义联结**
```python
# 从baseline数据中提取实体和关系

- Chip (1000个)

  - 物理损伤的芯片 → 翘曲度高、引脚平整度差
```

#### **子任务3: 序贯检测流程动态在线优化**
```python
# MDP建模

# 状态空间 (State):
- 已测试过的指标集合: {"pin_flatness", "vdd_voltage", ...}
- 已测试结果: {"normal", "suspicious", "fail"}

# 动作空间 (Action):
- "run_next_test": 继续测试下一个指标
- "terminate_pass": 提前判定为"正常品"，不再测试
- "terminate_fail": 提前判定为"故障品"，不再测试

# 奖励函数 (Reward):
- 正确判定 +1.0
- 错误判定 -1.0 (漏检或误检)
- 每多测一个指标 -0.1 (鼓励减少测试时间)

# PPO训练：
- 目标：最大化 (正确率 - 测试时间成本)
- 输入：当前状态 + 已有测试结果
- 输出：最优的下一步动作

# 结果示例：
- 对于"正常品"：平均测试15项指标即可判定 (原来28项)
- 对于"故障品"：平均测试12项指标即可判定 (早期诊断)
- 整体测试时间减少40% (但准确率不低于99%)
```

## 四、如何加载和处理数据

### 4.1 Python示例代码

```python
import pandas as pd
import numpy as np

# 加载三个文件
baseline_df = pd.read_csv('chip_baseline_data.csv')
aging_df = pd.read_csv('chip_aging_curves.csv')
failure_df = pd.read_csv('chip_failure_labels.csv')

print(f"基础数据: {baseline_df.shape}")      # (1000, 28)
print(f"老化曲线: {aging_df.shape}")        # (90424, 7)
print(f"失效标注: {failure_df.shape}")      # (100, 6)

# 筛选故障品
failure_chips = baseline_df[baseline_df['failure_status'] == 0]
print(f"故障品: {len(failure_chips)}")       # 100

# 查看某个故障品的完整信息
chip_id = failure_chips.iloc[0]['chip_id']
chip_baseline = baseline_df[baseline_df['chip_id'] == chip_id]
chip_aging = aging_df[aging_df['chip_id'] == chip_id]
chip_failure = failure_df[failure_df['chip_id'] == chip_id]

print(f"\n芯片 {chip_id} 的信息:")
print(f"基础参数: {chip_baseline.iloc[0]}")
print(f"老化曲线长度: {len(chip_aging)} (老化 {chip_aging['aging_hour'].max()} 小时)")
print(f"失效信息: {chip_failure.iloc[0]['failure_phenomenon']}")

# 提取特征向量用于机器学习
physical_features = baseline_df[[
    'pin_flatness_deviation_um',
    'solder_pad_oxidation_percent',
    'package_scratch_depth_um',
    'warpage_um',
    'pin_coplanarity_percent',
    'package_size_deviation_percent'
]]

electrical_features = baseline_df[[
    'vdd_voltage', 'idd_static_ma', 'idd_dynamic_ma',
    'voh_voltage', 'vol_voltage', 'vih_voltage', 'vil_voltage',
    'iil_leakage_ua', 'iol_leakage_ua',
    'setup_time_ns', 'hold_time_ns', 'propagation_delay_ns',
    'rise_time_ns', 'fall_time_ns', 'fmax_mhz'
]]

labels = baseline_df['failure_status']

# 机器学习模型训练示例
from sklearn.ensemble import RandomForestClassifier

X = pd.concat([physical_features, electrical_features], axis=1)
y = labels

model = RandomForestClassifier(n_estimators=100)
model.fit(X, y)

# 特征重要性分析
feature_importance = pd.DataFrame({
    'feature': X.columns,
    'importance': model.feature_importances_
}).sort_values('importance', ascending=False)

print("\n特征重要性排序:")
print(feature_importance.head(10))
```

### 4.2 多参数组合分析

```python
# 识别隐性故障的多参数组合

# 噪声余裕不足型故障品
noise_margin_insufficient = baseline_df[
    baseline_df['failure_mode'] == 'noise_margin_insufficient'
]

# 计算噪声余裕 (Noise Margin)
baseline_df['output_swing'] = baseline_df['voh_voltage'] - baseline_df['vol_voltage']
baseline_df['input_swing'] = baseline_df['vih_voltage'] - baseline_df['vil_voltage']
baseline_df['noise_margin'] = np.minimum(
    baseline_df['output_swing'] - baseline_df['input_swing'],
    baseline_df['input_swing']
) / baseline_df['vdd_voltage']

# 对比
normal_nm = baseline_df[baseline_df['failure_status'] == 1]['noise_margin'].mean()
failure_nm = baseline_df[baseline_df['failure_status'] == 0]['noise_margin'].mean()

print(f"正常品平均噪声余裕: {normal_nm:.3f}")
print(f"故障品平均噪声余裕: {failure_nm:.3f}")
print(f"差异: {(1 - failure_nm/normal_nm) * 100:.1f}%")
```

## 九、常见问题 (FAQ)

**Q1: 这些合成数据与真实数据的差异有多大？**  
A: 合成数据是根据工程统计学原理和实际失效机制设计的。物理约束（如VOH>VOL）和参数分布都符合真实芯片特性。但长期来看，需要与真实数据混合训练以提高准确性。

**Q2: 可以直接用这些数据训练生产模型吗？**  
A: 不建议。这些数据用于"概念验证"和"算法框架测试"。在真实环境中部署前，需要与企业的实际数据进行混合训练和验证。

**Q3: 为什么要生成时间序列（老化曲线）？**  
A: 时间序列能捕捉到"隐性故障的加速劣化现象"。只看初始参数会漏掉那些初始正常但动态劣化快的芯片。

**Q4: 如何扩展到更多样本？**  
A: 修改 generate_synthetic_data.py 中的 `n_chips` 参数。可以轻松生成10,000个甚至100,000个样本。
