## 三、合成数据生成的理论依据

### 3.1 数据分布设计原则

**原则1: 物理约束一致性**
```
✓ 生成时遵循的约束:
  - VOH ≥ VDD - 0.3V (输出高电压应接近电源)
  - VOL ≤ 0.3V (输出低电压应接近GND)
  - VIH > VIL (逻辑1判断电压 > 逻辑0判断电压)
  - VIH > VOH (要能被驱动)
  - VIL < VOL (要能被驱动)
  - Pin flatness, warpage 等物理参数独立但合理
```

**原则2: 工程统计分布**
```
✓ 使用的分布类型:
  - 正态分布 (高斯): VDD, VOH, VOL, 频率等(中心集中)
  - 指数分布: 功耗、漏电流等(长尾特性,罕见高值)
  - 均匀/混合分布: 芯片型号、生产日期等
  
  示例:
  - VDD ~ Normal(μ=3.3V, σ=0.08V) 限制在[3.135, 3.465]V
  - idd_dynamic_ma ~ Normal(μ=120mA, σ=40mA) 限制在[50, 200]mA
  - pin_flatness ~ Normal(μ=20μm, σ=8μm) 限制在[0, 50]μm
```

**原则3: 故障模式工程合理性**
```
✓ 四种故障模式的生成机制:

1. 物理损伤型 (20% of failures)
   └─ 增加: 引脚平整度偏差 +15~30μm
   └─ 增加: 翘曲度 +50~100μm
   └─ 增加: 划痕深度 +30~60μm
   └─ 现象: 焊接不良、断裂、脱焊

2. 噪声余裕不足型 (30% of failures)
   └─ 降低: VOH -0.15~0.3V
   └─ 升高: VOL +0.15~0.3V
   └─ 升高: VIH +0.1~0.2V
   └─ 降低: VIL -0.1~0.2V
   └─ 现象: 逻辑摆幅压缩,高频错误,EMI敏感

3. 热积累型 (25% of failures)
   └─ 升高: 动态功耗 +50~100mA
   └─ 增加: 翘曲度 +40~80μm
   └─ 设置: 老化时长=168小时
   └─ 增加: 划痕深度 +20~40μm
   └─ 现象: 功耗螺旋,热失控,关闭保护

4. 多参数叠加型 (25% of failures)
   └─ 升高: 建立/保持时间 +2~4ns
   └─ 升高: 传输延迟 +3~6ns
   └─ 降低: VOH -0.1~0.2V
   └─ 升高: VOL +0.1~0.2V
   └─ 升高: 动态功耗 +30~60mA
   └─ 现象: 时序违规,功能失效,边界工作

这些模式完全对应现实中的芯片失效机制！
```

### 3.2 参数漂移模型

**正常品的参数漂移** (老化过程中):
```
线性/缓慢漂移 (Degradation ≈ 0.5% ~ 3%)

示例:
VDD: 3.369V → 3.368V (漂移 -0.03% 在24小时内)
VOH: 2.966V → 2.965V (漂移 -0.03%)
延迟: 4.669ns → 4.670ns (漂移 +0.02%)

特征: 稳定,可预测,符合寿命规律
```

**故障品的参数漂移** (热积累型为例):
```
加速/非线性漂移 (Degradation ≈ 5% ~ 20%)

示例:
VOH: 2.966V → 2.940V (漂移 -0.88% in 168小时, 加速)
VOL: 0.077V → 0.130V (漂移 +69% in 168小时, 剧烈)
延迟: 4.669ns → 5.200ns (漂移 +11.4%)

特征: 非线性,加速恶化,最终雪崩
```

## 七、风险与对策

### 7.1 合成数据相关风险

| 风险 | 表现 | 对策 |
|-----|------|------|
| 模型过度拟合合成数据 | 在真实数据上准确率下降 | ✓ 预留验证集 ✓ 早期与企业数据混合 |
| 合成数据遗漏某些故障模式 | 真实中出现的故障无法识别 | ✓ 持续补充故障模式 ✓ 增量学习 |
| 参数分布与现实偏差 | 模型判断边界错位 | ✓ 逐步用真实数据校准 |

### 7.2 数据融合风险

| 风险 | 表现 | 对策 |
|-----|------|------|
| 真实数据质量差 | 污染混合训练集 | ✓ 严格的数据质量检验 |
| 企业数据获取困难 | 无法完成融合 | ✓ 保持纯合成模型可用 |
| 数据分布差异大 | 融合效果不理想 | ✓ 使用领域自适应技术 |
