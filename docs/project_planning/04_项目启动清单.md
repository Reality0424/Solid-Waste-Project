### 2.2 数据验证 (Day 1)

**运行检验脚本** (生成数据验证报告)
```python
# 脚本: verify_synthetic_data.py

import pandas as pd

# 加载数据
baseline = pd.read_csv('synthetic_data/chip_baseline_data.csv')
aging = pd.read_csv('synthetic_data/chip_aging_curves.csv')
failure = pd.read_csv('synthetic_data/chip_failure_labels.csv')

# 数据形状验证
print(f"Baseline: {baseline.shape}")  # 应为 (1000, 28)
print(f"Aging: {aging.shape}")        # 应为 (90424, 7)
print(f"Failure: {failure.shape}")    # 应为 (100, 6)

# 数据一致性验证
assert set(failure['chip_id']).issubset(set(baseline['chip_id']))  # 失效品应在baseline中
assert set(aging['chip_id']).issubset(set(baseline['chip_id']))    # 老化数据应在baseline中

# 物理约束验证
assert all(baseline['voh_voltage'] > baseline['vol_voltage'])
assert all(baseline['vih_voltage'] > baseline['vil_voltage'])
assert all(baseline['vdd_voltage'] > baseline['voh_voltage'])
assert all(baseline['vol_voltage'] >= 0)

print("✓ 所有数据验证通过!")
```

## 九、常见问题快速索引

**Q: 合成数据能直接用于生产吗?**  
A: 不能。这是MVP验证用的。生产前必须与企业真实数据混合并通过验证。

**Q: 如果企业一直无法提供数据怎么办?**  
A: 方案已备好。基于合成数据的模型也可以部署,但精度会有限制(约88-90%)。需要持续增量学习。

**Q: 四个子任务能并行进行吗?**  
A: 建议顺序推进,因为后续任务依赖前期任务的输出。但可以适度并行 (如子任务2可在子任务1完成50%时启动)。

**Q: 人才培养指标(≥2名)如何满足?**  
A: 通过学位论文(硕士) 或 技术报告(博士/工程师)。需要提前与导师沟通。

**Q: 专利/论文的创新点在哪里?**  
A: "多参数叠加隐性故障识别" 和 "强化学习动态检测优化"。这两个是项目的核心创新。
